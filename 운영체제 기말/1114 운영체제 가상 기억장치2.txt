1114 운영체제 가상 기억장치2

연관사상
- 직접사상과 달리 메인memory 이용X, 연관기억장치associative memory(CAM)사용
* CAM : 주소를 이용하지 않고, hash map처럼 key를 이용 (search속도가 빠름)(cache)
- 블록번호대신 key인 page number을 사용 (직접사상은 블록번호를 사용해 PMT(테이블)을 찾아감)

- 단점 : 가격이 비쌈


혼합사상
- 직접사상과 연관사상 혼합(연관사상 단점을 줄이면서 장점만)
- CAM에 PMT중 사용하는 정보만 올려놓음(프로그램전체가아닌 필요한 블록만 올려놓는 가상기억장치처럼)
즉, 작은 PMT가 됨(단, 블록번호를 page number로 바꿈)
*프로그램의 지역성
: 한번 접근된 부분은 계속 다시 접근됨, 그의 인접부분이 접근될 가능성이 높은 성질
- CAM에 적재되어있지 않아서 PMT에서 접근하여 사용한 부분을 블록번호를 page number(key)로 바꿔 그대로 CAM에다 저장

- 과정 :
1. 해당 프로세스의 PMT정보가 CAM에 올라와있는지 확인
1-1. 안올라 와있으면 직접사상기법으로 페이지프레임번호(실주소)를 인출, CAM에 PMT엔트리 적재
2. CAM에서 존재비트 검사후 페이지프레임번호 인출
=> 사용하려는데 먼저 CAM에 접근하고 CAM에 적재되어있지 않은 부분을 PMT에 직접 가서 꺼내와 사용하고 CAM에다 저장해놓고 반복
* 찾으려는 페이지번호가 CAM에 들어있는 경우 hit (or miss)
* hit확률 : hit ratio 

[11주차 가상 기억장치2 24:30 문제]
페이지번호 : 블록번호(가상주소의)
변위 : 페이징시스템에서의 실주소(PMT의)(block map table)
0011이 있으면 00부분이 블록번호(가상주소의 논리적주소), 11부분이 변위(가상주소의 변위)
블록번호인 00로 PMT(또는CAM)에 접근하여 실주소로 바꿔야함

-> 0101 0101110010는 가상주소, 앞부분은 블록번호, 뒷부분은 변위임
0101은 5, 5를 따라가서(CAM의 페이지번호)  4(CAM의 프레임번호)를 가져옴
페이지번호인 5를 프레임번호인 4로 바꿈
이진수로 바꾸면 0100 0101110010이 됨
