컴퓨터시스템구성
Hardware - Operating System - Application Programs

*Kernel : 항상 메모리에 올라와 있는 컴퓨터 부팅시 작동시키는 부분



OS의 발달과정 _os의 기능을 알기 위해

*프로그램을 실행하는데 OS가 없다면 직접해야할 것
1. bootstrap loader카드(부팅)를 memory에 load(부팅파일 카드를 컴퓨터에 읽히기)
2. compiler카드를 memory로 load(컴파일러 프로그램 카드를 컴퓨터에 읽히기)
3. source program카드를 compiler로 실행해서 binary code로 변환(실행)
4. binary code카드를 memory로 load (binary code 카드를 컴퓨터에 읽히기)
5. 프로그램에 입력할 data를 올려놓고 실행 (실행)
-> 1세대 OS => 컴퓨터에 카드 적재하고 실행버튼 누르고 하는 일들을 일괄처리
(비슷한 요구 조건을 갖는 여러 작업을 매크로 처럼 만드는 기법)



문제
Q. 다음 설명에 해당하는 운영체제 성능평가 기준은?
{컴퓨터 시스템 내 한정된 각종 자원을 여러 사용자가 요구할때, 얼마나 신속하고 충분히 지원해줄 수 있는지}
A. Availability

Q. 운영체제의 성능평가 요인중 다음 설명에 해당하는 것은?
{이것은 컴퓨터 시스템 내의 한정된 각종 자원을 여러 사용자가 요구할 때, 얼마나 신속하고 충분히 지원해 줄 수 있는지의 정도이다. 이는 사용 가능한 하드웨어 자원의 수나 다중 프로그래밍 정도 등의 요소가 좌우하는 것으로 같은 종류의 시스템 자원수가 많을 경우에는 이것이 높아질 수 있다.}
A. Availability

Q. 운영 체제의 성능을 판단 할 수 있는 요소로 가장 거리가 먼 것은?
1. 처리능력 2. 비용 3. 신뢰도 4. 사용가능도
A. 2.비용
